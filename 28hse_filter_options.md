## HTML Structure Reference

**Filter Menu Container:**
- Main container: `#mainMenuDiv`
- Filter items: `<a class="item withChildren">` or `<a class="item">`

**Data Attributes:**
- `data-value`: Internal value for the option
- `data-id`: Unique identifier (format: "number" or "number-number")
- `data-config`: JSON configuration string containing:
  - `allowMultipleSelect`: boolean
  - `deepOfmenuItems`: number (menu depth levels)
  - `allowCustom`: boolean
  - `allowSelectAll`: boolean
  - `onlyAllowSelectSameLayer`: boolean
  - And other configuration options

**Example HTML Structure:**
```html
<a href="https://www.28hse.com/buy/a1" 
   class="item withChildren" 
   data-value="hk" 
   data-id="2"
   data-config="{...JSON config...}">
    香港島
</a>
```

## Investigation Notes

**Accessibility Tree Snapshot Shows:**
- All filter links are present in the DOM
- Some links may not be immediately visible until parent category is clicked
- Multi-level navigation: Main category → Sub-category → Specific options
- Maximum menu depth observed: 3 levels (e.g., District → Region → Sub-district)

**Selector Patterns:**
- Filter items use class `item` with optional modifier `withChildren`
- Links have both `data-value` and `data-id` attributes for identification
- Some filters have direct href URLs (e.g., `/buy/a1` for Hong Kong Island)
- Other filters use JavaScript interactions (href="" or data-parent attributes)

**Behavioral Notes:**
- Clicking a main region (香港島, 九龍, 新界) reveals sub-districts
- Clicking "不限" (No limit) typically resets that filter category
- "自定" (Custom) options likely open custom range input dialogs
- Some filters may be interdependent (e.g., area type affects size range interpretation)
